HOW-TO-GUIDE FOR STUDENTS

3rd Year Project 2021-2022

BY MATTHEW CHAU, x1n32, 1900 4815

Special Thanks to my Supervisors: Fiona Polack and Marco Ortolani



What is on this webpage?

A General guide on GitHub and git commands.

Links to some java code to practice the basics, and learn about Unit Testing with JUnit


Aims and Objectives

To create a usable and understandable skills-aid demo for Keele Computer Science students to showcase the collaboration features of Collaborative Development Tools/ Version Control Systems that could be used for all programming and design modules.

How to get there:

  1. Research what students may need to learn that are relevant to their studies
  2. Research relevant studies that could help improve the quality and usability of the tutorials
  3. Create an information/skills page using GitHub (Pages/Classroom).
  4. Create two-to-three meaningful, but easy-to-implement Use Case scenarios (increasing in difficulty from beginner level) demos showcasing a specific topic in programming, where errors are purposely produced to showcase Unit Testing and Continuous Integration, so that students can learn from mistakes via meaningful error messages and so they can learn by doing.
  5. Create a questionnaire/survey to examine the quality, usability, and ethics of the project.

What I have done so far:


  1. Created a new Repositotry on GitHub
  2. Copied link of my address and cloned it to my computer using the clone command on the terminal: 'git clone https://github.com/x1n32/How-to-GitHub.git' (without the quotes)
  3. On the terminal, opened VSCode (code .) to create and edit a index file.
  4. Checking 'status' on the terminal (git status)
  5. 'addding' (git add .)
  6. 'commiting' (git commit -m inital commit)
  7. 'pushing' (git push origin main) my changes to the main branch
  8. Needed to generate a personal access token for VSCode to sign in instead of password: Go on GitHub, settings, developer settings, and Personal Access Token. Generate.
    9. Created another branch to test out adding css (git checkout -b Styling)
  9. Created a css document and linked it on href in the index.html file
  10. Adding those changes with git add .
  11. git commit -m styling
  12. git push origin Styling
  13. Merged Styles branch to Main Bracnh on GitHub via Pull Request
