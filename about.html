What is GitHub?

Git is a collaborative open-source version control system

This means a history of pushed changes make it easy to rollback/rollforward to easily make changes.

This also make it easy to work with others as you can share code to work on the same project

It makes use of branches, where you make 'alternate timelines' to test out code before pushing it to the main code.

Anyone can create repositories where it gets stored on GitHub to back-up code or to share!

You can use GitHub Pages to load html files like this.

General Workflow:

  1. Create or find a Repositotry that you want to work on
  2. Copy link and clone it on a command line (git clone addressName) or download the files
  3. Check 'status' on the terminal (git status)
  4. Once you made any changes that you want to add to the main file, add your changes to the staging area:
  5. 'addding' all files: (git add .)
  6. 'commiting' with a meaningful message to the changes (git commit -m "inital commit")
  7. 'pushing' (git push origin main) my changes to the main branch
    8. If you want to test your changes by making another branch: (git checkout -b branchName)
  8. After you make more changes, to the same from git add. then you can push your changes to the new branch (git push origin branchname)
  9. A Pull Request will be available to the host to accept those changes.
